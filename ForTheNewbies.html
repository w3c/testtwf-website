W3C's HTML Testsuite Effort

Vendors want to know if they've implemented things correctly, and W3C wants to know how much has really been implemented out there. Lots of people want to know how to bridge the gap between specs and “tag soup.” To answer these questions, W3C aims to create a comprehensive testsuite of the web platform, and it needs your help!

Here's the Status

GitHub hosts all of the HTML Working Group's current tests (11,907 and counting), both approved and pending. The following specs are the ones for which complete test suites are critically needed:

  • HTML 5, CR
  • HTML Canvas 2D Context, CR
  • HTML Microdata, CR

Here is a snapshot of the test coverage (approved tests only) (would prefer CR version that also includes pending): http://w3c-test.org/html-testsuite/master/tools/coverage/

It's just the beginning!

Got Bugs?

W3C uses Bugzilla to keep track of bugs and ideas. If you haven't already, open a new W3C BugZilla-specific account so that you, too, can file bugs.

Test-Specific Bugs
If you think an approved test needs to change, file a bug here.
Test Harness
If your test isn't behaving as you think it should, and you think it's due to the test harness engine, file a bug here.
Overall Testsuite Organization and Posted Test Results
If you're having trouble with the big-picture side of the testsuite, file a bug here.
Spec Bugs
Or, if you find that the real world doesn't match a spec's expectation in some significant way, file a bug on that spec:
  • HTML 5
  • Canvas
  • Microdata
